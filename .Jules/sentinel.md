## 2024-05-24 - [SSE Stream Authorization Fix] **Vulnerability:** Unauthenticated users could subscribe to any SSE channel (e.g., `dm:userA_userB` or `expedition:123`) by passing the channel name as a URL parameter, allowing unauthorized access to private real-time messages. **Learning:** The SSE route (`src/app/api/comms/stream/route.ts`) verified the user was logged in but failed to verify if the user had authorization for the specific `channel` requested before starting the stream connection. **Prevention:** Always validate user authorization against the specific resource (channel, topic, expedition) they are attempting to access or subscribe to, especially in real-time or streaming endpoints.
## 2024-05-28 - [Authorization Bypass via Substring Matching] **Vulnerability:** DM channel access checks used loose `.includes(userId)` and Prisma `contains`, allowing users whose IDs were substrings of valid participants to access private channels. **Learning:** Channel format validations (`dm:{userId1}_{userId2}`) inherently leak substrings. String operations must strictly delineate and parse specific user IDs instead of loosely checking the entire string. **Prevention:** Always split delimited strings into discrete components (`.split('_')`) or use exact-match boundary conditions (e.g., `startsWith`, `endsWith`) before applying validation checks.
